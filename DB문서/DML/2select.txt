사용자가 입력한 데이터는 언제라도 조회가 가능하다

조회하기를 원하는 컬럼명을 select 다음에 콤마 구분자(,)로 구분하여 나열하고
from 다음에 해당 컬럼이 존재하는 테이블명을 입력하여 실행한다

[]: 선택사항
*표현식
select [all/distinct] 보고 싶은 컬럼명, 컬럼명, ,,,
from 해당 컬럼들이 있는 테이블명;

기본값: all
distinct: 중복된 데이터가 있는 경우 1건으로 처리해서 출력한다


*데이터 딕셔너리 tab
계정에 존재하는 테이블을 조회하는 방법
테이블의 정보를 알려주는 데이터 딕셔너리 : tab


*데이터 딕셔너리
데이터베이스와 관련된 정보를 제공한다

*테이블 구조를 살펴보기 위한 desc[ribe]
desc 테이블명


*NULL 타입?
특징
-널 값은 아직 정의되지 않은 값으로 0 또는 공백과 다르다. 공백은 하나의 문자이다
-테이블을 생성할 때 NOT NULL 또는 PRIMARY KEY로 정의되지 않은 모든 데이터 유형은
널 값을 포함할 수 있다
-널 값을 포함하는 연산의 경우 결과 값도 널 값이다.
모르는 데이터에 숫자를 더하거나 빼도 결과는 마찬가지로 모르는 데이터인 것과 같다
-NULL 값의 대상이 숫자 유형 데이터인 경우는 주로 0(ZERO)으로, 문자 유형 데이터인 경우는
블랭크보다는 'x'같이 해당 시스템에서 의미 없는 문자로 바꾸는 것도 고려해 보자


*Alias알리아스(별칭)  부여하기
조회된 결과에 일종의 별명(Alias)을 부여해서 컬럼 레이블을 변경할 수 있다

별명에서 지켜야할 사항
-컬럼명 바로 뒤에 온다
-컬럼명과 alias 사이에 as, As 키워드를 사용할 수도 있다
-이중 인용부호""는 alias가 공백, 특수문자를 포함할 경우와 대소문자 구분이 
필요할 경우 사용된다


*CONCATENATION 연산자
자바에서 배운 + 연산자의 특징과 동일하다
무조건 문자열로 변환해서 이어준다
